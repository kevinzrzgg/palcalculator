## 1. 执行决定

建议：启动 PalCalculator 作为免费、无需登录的 Palworld 计算器中心。请勿在 MVP 中发送付款、订阅、帐户墙或付费门控。

为什么：
- PRD 明确表示“发布时付费 SaaS”不是产品类型，并且“请勿为 MVP 建立付款/订阅。”
- 用户意图是免费游戏实用程序：繁殖计算器、路线求解器、IV/统计计算器和被动规划器。
- 差异化应该来自快速路线优先的用户体验、当前的 Palworld 1.0 数据、信任层和可共享的结果，而不是货币化摩擦。
- MVP 转换是参与：工具完成、共享/复制、跨工具导航和信任页面可见性。

MVP商业姿态：
- 免费：YES，所有 P0 计算器无需登录即可使用。
- 付费：MVP 否。仅接受“支持者功能稍后推出”/候补名单副本。
- 广告：发布时 NOT；在流量验证之后考虑，切勿高于计算器输入。
- 附属机构：MAYBE 稍后在合规性审核后进行游戏/服务器托管推荐。
- 潜在客户开发：LOW 优先用于 MVP；仅当隐私副本准备就绪时，可选时事通讯/Discord/补丁警报注册。

## 5. 成本模型和安全上限

### 5.1 成本假设

P0 应设计为具有非常低的边际成本：
- 静态计算器 UI 和 JSON 数据通过 Cloudflare 页面/静态资产提供。
- 在可行的情况下首选客户端计算。
- 仅当路由求解对于客户端性能而言过于繁重时，才使用 Worker API。
- MVP 中没有服务器端保存文件上传或持久性 Palbox 存储。
- P0 中没有第三方付费 AI/API 依赖项。

由于确切的 Cloudflare 帐户计划、Workers 使用定价、分析提供商和存储选择尚未确认，因此本报告未断言最终的美元成本。它定义了单位成本驱动因素和实施移交所需的上限。

### 5.2 单位成本表

|单位| MVP 路径 |预期边际成本类别|所需的上限/控制|
|---|---|---:|---|
|页面浏览量 |静态 Cloudflare 页面路线 |非常低|缓存静态资源；保持 JS/数据包精简。 ||计算器开始|浏览器端UI事件|非常低|分析事件必须避免原始私人数据。 |
|育种对查找|静态JSON + 客户端函数|非常低|验证输入；默认情况下不需要服务器。 |
|反向父搜索 |静态JSON + 客户端函数|非常低|如果数据集允许，则在客户端进行过滤/排序。 |
|路线解决 |更喜欢客户端；如果需要，工人后备|如果工人较多，则低到中等 |最大拥有的 Pal 列表长度、最大代数、超时、缓存安全公共结果、速率限制滥用。 |
| IV/stat 计算 |客户端公式 |非常低|公式版本/注意事项；无外部 API。 |
|被动计划者外壳 |静态数据+客户笔记|非常低|不要计算不受支持的概率。 |
|分享结果 |编码 URL 状态首选 |非常低|无原始保存数据；默认情况下为 noindex 结果 URL。 |
|已保存的 Palbox/历史 |不在 MVP |存储 + 稍后的验证成本 |构建前的 OWNER_DECISION_REQUIRED；需要硬性保存项目上限。 |

### 5.3 免费层成本安全

如果 P0 保持静态/客户端较多，则免费套餐是安全的。唯一对成本敏感的风险是服务器端路由解决或未来存储的用户特征。

后端/前端的实施护栏：
- 从静态 JSON 和纯函数开始。
- 在引入 Worker API 之前进行基准路线求解。
- 如果使用 Workers，则返回持续时间桶和错误代码，但不记录原始拥有的 Pal 列表。
- 如果任何 API 端点接受拥有的 Pal 列表，则添加服务器端请求正文大小上限。
- 无需创建帐户即可保持所有 P0 功能可用。
