## 1. 执行产品决策

决定：BUILD 一个 Palworld 计算器中心，品牌为 PalCalculator。

不要将其定位为通用 `palcalculator` 完全匹配站点。上游研究发现对 `palcalculator` 的精确匹配需求较弱/嘈杂，但 Palworld 计算器对育种、1.0 育种、IV、统计数据、被动技能和路线规划的意图强烈。因此，PalCalculator 应该使用简短的品牌名称，而每个可索引的承诺都会使 Palworld 上下文变得明确。

一行定位：
对于想要在不使用电子表格工作的情况下繁殖或评估目标 Pal 的 Palworld 玩家来说，PalCalculator 是一款由粉丝快速制作的 Palworld 1.0 计算器中心，它将繁殖对、最短拥有的 Pal 路线、IV/统计检查和被动技能规划结合在一个移动优先工具中。

主要竞争楔子：
- 不仅仅是父对查找。
- 路线优先：“我有这些伙伴；到达我想要的伙伴/构建的最短路径是什么？”
- 信任第一：明确 Palworld 1.0 数据版本、来源/更新政策、公式警告和粉丝网站免责声明。
- 用户体验优先：首屏计算器输入、无帐户墙、可共享结果 URL 以及快速 Cloudflare 边缘交付。

## 2. 机会和证据总结

上游判决：BUILD Palworld 1.0 计算器中心；不要单独定位精确的 `palcalculator`。

研究的关键证据：
- Google 自动建议将机会标准化为 `pal calculator`、`pal calculator breeding`、`pal calculator palworld`、`palworld calculator`、`palworld calculator 1.0`、`palworld calculator breeding`、`palworld calculator iv`、`palworld calculator stats` 和`palworld calculator passive`。
- 关键字工具发现 `palworld 1.0 breeding calculator` 作为上升术语加上 `breeding calculator palworld`、`pal breeding calculator`、`pal analyzer` 和 `palworld save editor`。
- 竞争对手验证需求：Palworld.gg、Palpedia、Pal Routes、PalworldBreedCalculator.com、Game8、Wikily/Paldex、PalDB、Pal-Breed 和 XGamingServer 均涵盖重叠工具。
- 差距：许多工具解决一项孤立的任务； PalCalculator 可以通过将路线、育种、IV/stat、被动规划、共享链接以及对初学者友好的解释与当前 1.0 数据捆绑在一起来获胜。

下游 SEO 复制冻结的警告：
- 研究使用公共自动建议、Bing RSS、Steam API、关键字 API 和直接竞争对手元数据，因为配置的 Tavily 搜索/提取返回 401。在最终标题/元副本冻结之前，仍然建议手动实时 Google SERP 审查。

## 3. 产品类型

站点类型：混合计算器/工具中心。

主要类型：
- Palworld 的计算器实用程序网站。

次要类型：
- Pal 数据、被动式、公式和源/更新注释的数据查找外壳。
- 一旦数据质量得到验证，针对特定 Pal 任务的编程 SEO 页面。

不是：
- 通用 PAL/健身/航空公司/PayPal 计算器。
- 完整的 Palworld 维基。
- 将编辑器保存在 MVP。
- 推出时付费 SaaS。

主要 ICP：
高度专注的 Palworld 玩家正在规划繁殖结果或目标构建，尤其是那些知道自己想要的 Pal 并且需要从当前 Palbox 中获得最少可靠繁殖步骤的玩家。

### P0 用户任务

1. 计算两个父母 Pals 的孩子。
   - 触发点：用户有两个朋友并且想要了解后代。
   - 成功：子项、特殊组合标记和警告立即可见。

2. 查找目标好友的所有父对。
   - 触发器：用户想要Anubis/Jetragon/Orserk/等。并且需要有能力的父母。
   - 成功：使用过滤器对父母对进行排序，并直接链接到路线规划。

3. 求解从拥有的好友到目标好友的最短路径。
   - 触发器：用户拥有有限的 Palbox 并且想要最少的代数。
   - 成功：路线步骤、代数、缺少的成分以及复制/分享链接。

4. 估计 IV/stat 质量。
   - 触发器：用户拥有好友等级/统计数据，并想知道它是否值得保留/繁殖。
   - 成功：IV 范围/分数、预期统计范围和公式假设。

5. 规划被动技能继承，但不要过分承诺确定性。
   - 触发器：用户想要一个具有选定被动技能的目标伙伴。
   - 成功：候选父母/路线和 RNG/继承警告。

6. 验证数据新鲜度。
   - 触发因素：用户担心 Palworld 1.0 更新后计算器已经过时。
   - 成功：数据版本、更新日期和来源/更新策略可见。

## 5. 定位和价值主张

定位声明：
对于需要快速繁殖、路线、IV/统计和被动规划的 Palworld 1.0 玩家，PalCalculator 提供了一个基于浏览器的粉丝制作的计算器中心，可将目标 Pal 或当前 Palbox 转换为逐步路线和可共享的结果，而不像密集的 wiki 或单一用途计算器，让用户手动将繁殖、统计和被动信息拼接在一起。

价值层次：
1. 快速获得答案：首屏计算器 UI。
2. 知道下一步该做什么：路由步骤并复制/共享结果链接。
3. 相信结果：Palworld 1.0 数据版本、公式、假设、警告。
4.保持轻量级：无需登录，无需服务器端保存上传，移动优先。
5. 仅当每个页面具有真正的工具价值时才通过长尾页面进行扩展。

主要 CTA：
- 从目标伙伴开始。

二级 CTA：
- 计算繁殖对。
- 检查 IV/统计数据。
- 计划被动。
- 粘贴拥有的好友列表（浏览器本地，如果可用）。

## 7. NOT-DO 列表

1. 不要将 `palcalculator` 单独定位为 SEO 头部术语。
2. 不要构建亲本配对育种计算器的精简克隆。
3. 请勿将网站呈现为官方网站或使用看似官方的品牌。
4.未经许可，不得使用受版权保护的游戏美术/徽标/资产。
5. 不要发布过时的或来源不明的育种数据；无隐藏数据版本。
6. 不要过度承诺被动继承或精确的 IV 确定性，因为公式/RNG/rounding 会产生不确定性。
7. 不要将存档上传到MVP中的服务器；如果存在导入，则更喜欢浏览器本地解析和清晰的隐私副本。
8.不要将工具埋在长SEO文本下面。
9. 不要创建不包含具体计算器/用户任务的宽泛 wiki 页面。
10.不需要登录即可完成P0任务。
11. 不要为 MVP 建立付款/订阅。
12. 不要抓取/克隆竞争对手的数据作为唯一的事实来源。

## 9. 功能要求和状态

### 9.1 共享计算器 shell

输入：
- 带有自动完成、别名和空状态示例的朋友选择器。
- 数据版本选择器/标签，默认最新稳定的Palworld 1.0数据。
- 可选的拥有伙伴列表输入用于路线任务。

国家：
- 空：显示示例查询和首要任务。
- 加载：控件下的骨架/结果旋转器，而不是整个页面块。
- 成功：通过复制/共享、数据版本、警告可见结果。
- 无结果：解释原因并建议放宽过滤器或检查拼写。
- 无效输入：内联验证，没有页面重新加载。
- 数据不可用：显示 `[待确认]`/不可用状态而不是捏造的结果。

### 9.2 育种计算器

输入：
- 家长_朋友
-parent_b 朋友
- 可选过滤器：包括/排除特殊组合、数据版本

输出：
- 儿童伙伴
- 特殊组合标记（如果适用）
- 链接到反向目标对
- 链接到以孩子为目标的路线求解器
- 来源/更新说明

边缘情况：
- 同一个父母两次。
- 未知的朋友名字。
- 朋友不可繁殖或被数据排除。
- 特殊组合优先于普通配方。

### 9.3 反向父搜索

输入：
- 目标伙伴
- 可选过滤器：仅拥有、稀有、世代计数、仅特殊、排除传奇/特殊

输出：
- 父母配对列表
- 按可用性/简单路线/稀有性/名称排序
- 将每对链接到路线详细信息

边缘情况：
- 目标没有普通的对，但有特殊的组合。
- 非常大的结果集；分页/过滤。
- 数据冲突；显示警告。

### 9.4 繁殖路线求解器

输入：
- 手动选择拥有的好友列表或粘贴为类似文本/CSV 的列表
- 目标伙伴
- 可选约束：最大代数，排除好友，包括受支持的被动

输出：
- 最短路线/最少代数
- 逐步的育种计划
- 需要失踪的朋友
- 替代路线（如果有）
- 复制/分享 URL

边缘情况：
- 用户已经拥有目标。
- 没有限制内的路线。
- 拥有的列表为空：回退到一般父对。
- 朋友名字重复/拼写错误。
- 平局：显示替代方案和平局规则。

### 9.5 IV/统计计算器

输入：
- 帕尔
- 级别
- 观察HP/攻击/防御
- 灵魂/聚光星/被动/信任或数据支持的其他已知修饰符

输出：
- 估计 IV 范围/分数
- 预期统计范围
- 公式假设
- 舍入或缺失修饰符适用的置信度/警告

边缘情况：
- 缺少修饰符会产生广泛的范围。
- 不可能的统计值。
- 不支持的 Pal/data 版本。
- 不确定的公式必须明确。

### 9.6 被动计划者

输入：
- 目标伙伴
- 所需的被动技能
- 可选拥有的朋友/候选父母

输出：
- 候选路线注释
- 继承警告和 RNG 警告
- 建议的下一步行动：选择父母、繁殖路线或验证数据

边缘情况：
- 被动组合不可用/不支持。
- 太多想要的被动。
- 数据不能保证概率。

## 10.数据合约需求完整实现属于后端/数据阶段，但 PRD 需要这些域对象和策略。

所需数据实体：
- Pal：id、slug、显示名称、别名、元素、稀有度、可繁殖标志、数据版本。
- BreedingPair：parent_a_id、parent_b_id、child_id、combo_type、源、data_version。
- SpecialCombo：父约束、child_id、解释、源、data_version。
- PassiveSkill：id、名称、类别、效果、兼容性说明、来源、数据版本。
- StatFormula：pal_id/基本统计数据、支持的修饰符、公式版本、警告、来源。
- RouteResult：target_pal_id、owned_pa​​l_ids、步骤、世代、missing_pals、替代方案、约束、data_version。
- ShareState：以 URL 或短 ID 安全编码的路由/构建参数。

数据政策：
- 在每个工具页面上显示最新数据版本和最后更新日期。
- 将源/更新策略保留在 `/data-sources/`。
- 如果数据集大小允许，首选确定性静态 JSON 作为 P0。
- 如果客户端性能不足，仅使用 Workers 进行路由求解。
- 不要在 MVP 中发送保存文件/服务器端 Palbox 数据，除非所有者批准隐私/安全设计。
- 绝不捏造缺失数据；显示不可用/警告状态。

## 11. 非功能性需求

性能：
- 移动 LCP 生产版本目标低于 2.5 秒。
- 对于常见情况，计算器交互应该感觉即时；如果 >500 毫秒，路线求解应显示进度。
- Cloudflare 页面静态优先；仅在必要时进行 API/计算的工作人员。

辅助功能：
- 键盘可用的表格和自动完成。
- 每个输入的标签。
- 颜色并不是稀有/警告的唯一信号。

SEO/可转位性：
- 每个可索引路线上的规范标签。
- 站点地图仅包括可索引的规范页面。

隐私/合规性：
- 页脚和条款上的粉丝制作/非官方免责声明。

## 12. 转化目标

MVP 转化定义是参与度，而不是付款。

主要目标：
- 用户完成计算器任务。
- 用户复制或共享结果。
- 用户从一个计算器单击到另一相关任务。

第一个验证窗口的建议可衡量目标：
- 核心页面的工具启动率：>= 工具页面上的会话的 35%。
- 开始后工具成功率：>= 70% 用于育种和 IV/stat 计算器；如果约束很复杂，则路径求解器 >= 50%。
- 复制/共享操作：>= 成功路线/构建结果的 8%。
- 减少跳出：主页用户点击工具或启动目标输入10秒内>=25%。- 数据信任：`/data-sources/` 从每个工具页面链接，并且在 QA 期间不会收到未解决的新鲜度投诉。

商业方向：
- 免费启动。
- 仅在吸引力之后考虑广告，切勿高于计算器输入。
- 合规性审核后，可以稍后探索游戏/服务器托管的附属链接。
- 在重复使用证明保存的 Palbox/路线历史值之前，不要添加订阅/付费专区。

## 13. 验收标准

PRD 登机口验收：
- 存在规范的 URL 计划并包括索引/无索引决策。
- 用户任务是明确的并且与 Palworld 计算器意图相关。
- MVP 路由合约存在于 /root/projects/palcalculator/artifacts/route-contract.md 中。
- NOT-DO 列表可防止通用 PAL/宽 wiki/范围蔓延。
- 验收标准是可衡量的，包括边缘情况。

MVP 实施的产品验收：
1. `/`、`/breeding-calculator/`、`/breeding-route-calculator/`、`/iv-calculator/`、`/stats-calculator/`、`/passive-skill-calculator/`、`/palworld-1-0-breeding-calculator/`、`/data-sources/`、`/privacy/` 和`/terms/` 渲染没有 404。
2. 每个可索引工具页面都有一个规范的 URL、H1、标题/元、首屏计算器入口点、数据版本标签和粉丝站点免责声明路径。
3. 育种计算器返回有效父对的子代，并针对错误输入显示明显的无效/无结果状态。
4. 反向目标搜索返回有效目标的父对或无结果解释。
5. 路线求解器处理拥有列表为空、目标已拥有、无路线以及至少一条成功的多步路线。
6. IV/统计计算器可处理有效输入、不可能的统计数据、缺失的修饰符以及带有明确警告的不受支持的数据。
7. 被动规划器不主张确定性继承；不支持的组合显示警告。
8. 无需登录即可共享/复制结果，并且不会泄露私人数据。
9.移动视口无需水平滚动或隐藏控件即可完成每个P0任务。
10. 以非秘密形式触发 page_view、tool_start、tool_success、tool_error、复制/共享的基本分析事件。
11. 站点地图和机器人排除共享/结果状态 URL，除非稍后有意推广。
12. 启动不需要受版权保护的游戏艺术/徽标。
